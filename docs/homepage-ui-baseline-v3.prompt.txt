Use case: ui-mockup
Asset type: high-fidelity desktop website homepage design mockup, spacious section-by-section landing page

Primary request:
Create a refined third-version visual UI design for CodeToken AI, a Codex / Claude Code API gateway and token relay service. Improve the previous concept by making the page more spacious, premium, and atmospheric. The whole page must feel less crowded and less compressed.

Key change from previous version:
The layout should be large and calm: roughly one desktop screen height per major section. Each region should breathe, with bigger hero blocks, larger data visuals, stronger whitespace, and fewer small cards. The design should feel grand, professional, and trustworthy, not dense or cramped.

Header requirement:
Use the header/navigation style from the first version: a clean glassy white business header, steady horizontal navigation, clear brand on the left, nav items centered, compact right-side action/status. It should feel polished and stable, not futuristic or cluttered.

Audience:
Developers, AI coding tool users, AI teams, automation-heavy users, and API resellers who need stable long-running coding sessions, multi-channel failover, fast response, and lower model access cost.

Overall design direction:
Premium light enterprise SaaS. Spacious, confident, business-professional, technical, credible, actively operated. Less text, more large visual proof. Use section-level scroll choreography: each major block could occupy one desktop viewport, with a clear visual anchor, subtle background telemetry, large charts, and elegant transitions between sections.

Canvas and framing:
Tall desktop landing-page design mockup. Wide 1600px-style layout. Show multiple large vertical sections, but do not cram every detail into one screen. Make every section feel like its own screen: hero, real operations curve, scenario/routing proof, integrations/conversion. Keep bottom visible. Use large margins and generous vertical spacing.

Required page structure:
1. Header: first-version style, glassy white, clean business navigation.
2. Hero screen: large left headline and CTAs, right side premium gateway product visual. Big, cinematic but still clean. Show Codex / Claude Code request entering one gateway and routing to healthy/degraded/standby upstream channels.
3. Real Operations Curve screen: large status analytics section with group switching tabs because each group has different channel access status:
   - "全部"
   - "Codex 专用"
   - "Claude Code"
   - "高速组"
   - "低价组"
   This section should have a large chart as the hero object, not many tiny widgets. Show success rate, latency, request volume, and protection events in a calm professional dashboard style.
4. Routing and cost screen: show one Base URL and one key on the client side, smart routing by model/group/health/cooldown/cost in the middle, upstream channel pool on the right. Use a large clean flow diagram.
5. Scenario / ecosystem screen: large scenario panels for long coding sessions, upstream rate-limit transfer, team cost control, IDE/script automation. Keep them visual and spacious.
6. Final CTA screen or bottom band: concise confidence message and two actions.

Visual requirements:
- Make the page more spacious and premium than the previous version.
- Do not pack too many widgets into each region.
- Use fewer, larger, stronger visual components.
- Give each section strong vertical breathing room, like one desktop screen per section.
- Reduce text density: avoid paragraph-heavy copy and tiny labels.
- Use large charts and diagrams, not many equal cards.
- Make the Real Operations Curve section especially polished: group tabs are clear, chart is dominant, metrics are large and readable.
- Use subtle scroll-effect cues: sticky navigation, scroll progress marker, layered telemetry lines, section fade/overlap, pinned chart panel, but keep it professional.
- Add relevant technical ambience through terminal strips, API traces, code-agent activity, route lines, channel status indicators, and cost-ratio chips.

Color palette:
White and cool gray base, teal/cyan identity accents, blue data accents, green healthy status, amber for cooldown/protection, red only for failure markers. Avoid purple gradients, dark-only hero, beige/brown themes, decorative blobs, bokeh, or random abstract art.

Avoid:
No crowded dashboard. No compressed long-page collage. No cartoon illustration. No stock photo. No decorative blobs or orbs. No generic card soup. No tiny admin tables. No weak narrow centered layout. No huge paragraph blocks. No dark hero block. No purple/blue gradient hero. No fake unreadable microtext. No messy overlapping UI. No bottom cropping.
