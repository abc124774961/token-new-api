# CodeToken AI User Console Design Style

## 定位

普通用户控制台采用全新的 `Aurora Enterprise Console` 设计语言，不沿用旧版控制台视觉。它是面向开发者和普通用户的 AI API 调用工作区，重点服务接入、令牌、日志、状态和费用管理。

参考效果图：

- `/Users/frode.luo/project/token-new-api/output/imagegen/navigation-redesign-user-console-v5-image2.png`

## 设计目标

- 与管理员后台属于同一套商业 SaaS 产品家族。
- 信息密度低于管理员后台，强调清晰、可信、易上手。
- 侧边导航只展示普通用户能力，不混入管理员后台菜单。
- 个人设置、安全设置、语言设置、退出登录等入口统一放入右上角用户菜单。
- 页面第一眼应让用户知道自己正在管理 API 接入与调用凭证。

## 视觉语言

整体风格为明亮、商业化、统一组件化的 AI 控制台。

- 背景：冷灰近白背景，避免纯白造成层级不清。
- 表面：珍珠白卡片与表格面板。
- 主文字：深石墨色，偏企业级、稳定可信。
- 次级文字：冷灰蓝，弱化说明信息。
- 品牌色：高精度青绿色 / 青蓝色，用于主按钮、选中态、关键图标。
- 辅助色：少量蓝色、橙色、绿色、红色，仅用于状态和趋势。
- 阴影：轻量柔和，不使用夸张浮层阴影。

## 布局结构

普通用户控制台使用三段式布局：

```text
深色品牌窄栏 + 白色业务侧栏 + 顶部工作区栏 + 主内容区
```

- 深色品牌窄栏用于承载 Logo、产品识别和折叠入口。
- 白色业务侧栏用于普通用户二级导航。
- 顶部工作区栏显示面包屑、通知、语言、用户信息。
- 主内容区以指标卡、工具栏、表格为基础。

推荐尺寸：

```text
品牌窄栏: 72px - 84px
业务侧栏: 260px - 300px
顶部栏: 72px - 82px
主内容 padding: 28px - 32px
卡片圆角: 10px - 12px
表格行高: 58px - 64px
```

## 导航结构

普通用户侧边栏只保留用户任务，不出现管理后台。

```text
概览
- 数据看板
- 服务状态

开发接入
- 操练场
- 令牌管理
- 使用日志
- 绘图日志
- 任务日志
- 聊天

费用中心
- 账户充值
- 套餐订阅
- 邀请有奖
```

当前选中项使用：

- 左侧 3px - 4px 青绿色竖向指示条。
- 浅青绿色背景。
- 青绿色图标和文字。
- 同级子项保持固定行高，避免内容跳动。

禁止在普通用户侧边栏中出现：

```text
管理后台
渠道管理
账号池管理
模型部署
盈利监控台
系统设置
个人设置
```

## 顶部用户菜单

个人相关能力统一放入右上角用户菜单。

```text
个人资料
安全设置
通知偏好
语言设置
进入管理员后台 仅管理员可见
退出登录
```

`个人设置` 不再作为侧边栏一级或二级菜单。

## 组件规范

### 指标卡

- 普通用户页面建议每行 4 个指标卡。
- 指标卡图标使用统一圆角色块。
- 数字使用较大字号，趋势信息放在底部。
- 不堆叠过多解释性文字。

### 表格

- 表格用于令牌、日志、账单等列表。
- 行高固定，视觉节奏稳定。
- 操作区使用简短文本或图标按钮。
- 状态统一使用圆角胶囊标签。

### 按钮

- 主按钮统一使用青绿色实心样式。
- 次按钮为白底描边。
- 危险操作使用红色文本或红色弱背景，不使用大面积红色按钮。

### 图标

- 图标必须统一线宽和圆角风格。
- 普通用户侧图标更轻，不使用过强管理感图标。
- 不混用多套图标视觉。

## 页面气质

普通用户控制台应更像开发者工作台：

- 少菜单。
- 少告警压迫感。
- 强调接入状态、额度、调用趋势。
- 让用户快速完成令牌创建、日志查看和充值订阅。

避免：

- 营销站导航。
- 大面积渐变背景。
- 装饰性光斑。
- 过多表格列。
- 管理员运营指标。
- 原始功能列表堆叠。
