# CodeToken AI Admin Console Design Style

## 定位

管理员后台采用全新的 `Aurora Enterprise Console` 设计语言，与普通用户控制台属于同一套商业 SaaS 产品家族，但信息密度和运营属性更强。它是面向管理员、运营和技术维护人员的独立后台，后续可按独立目录、独立路由、独立部署演进。

## 设计目标

- 管理员后台独立于普通用户控制台，不作为普通侧栏的一段菜单。
- 首屏优先服务运营监控、渠道稳定性、模型路由、商业收益、结算和治理。
- 与普通用户控制台共享统一视觉系统，但允许更高信息密度。
- 菜单、卡片、筛选器、状态标签、表格行高保持统一组件语言。
- 管理员一眼能看到风险、异常、预警和可操作入口。

## 视觉语言

整体风格为明亮、商业化、高密度但不压迫的企业级后台。

- 背景：冷灰近白。
- 表面：珍珠白面板与表格。
- 主文字：深石墨色，强调专业和稳定。
- 次级文字：冷灰蓝，用于 ID、说明、辅助数据。
- 品牌色：高精度青绿色 / 青蓝色，用于主操作、选中态、健康状态。
- 风险色：橙色表示预警，红色表示异常或严重风险。
- 图表线条：小型趋势图使用同一色系，不做夸张视觉装饰。

## 布局结构

管理员后台使用独立后台布局：

```text
Admin 顶部栏 + 管理员专用侧栏 + 运营指标区 + 高密度管理表格
```

推荐尺寸：

```text
业务侧栏: 280px - 320px
顶部栏: 72px - 82px
主内容 padding: 28px - 32px
KPI 卡片圆角: 10px - 12px
表格面板圆角: 10px - 12px
表格行高: 56px - 62px
筛选控件高度: 40px - 42px
```

管理员后台可以不显示普通用户控制台的深色品牌窄栏，但 Logo、顶栏、侧栏选中态、按钮和状态标签必须与普通用户控制台保持同源视觉。

当前选中项使用：

- 左侧 3px - 4px 青绿色竖向指示条。
- 浅青绿色背景。
- 青绿色图标和文字。
- 父级菜单保持展开态，箭头方向明确。

禁止在管理员侧边栏中出现：

```text

```

## 顶部栏

管理员顶部栏应强调后台身份与环境。

```text
CodeToken AI Admin
管理员后台
管理员后台 / 渠道运营 / 渠道管理
Pro 环境标识
全局搜索
告警中心
任务队列
语言
管理员账号
```

顶部栏不展示营销导航，也不展示普通用户的充值、套餐、邀请入口。

## 首屏信息

管理员首页或渠道管理页首屏优先展示运营与风险指标。

```text
今日请求
成功率
渠道预警
智能切换
结算异常
今日利润
```

指标卡要求：

- 每张卡保留统一图标块、主数值、趋势信息。
- 预警和异常卡使用橙色、红色，但面积要克制。
- 可使用小型趋势线，但不得喧宾夺主。

## 表格与运营区域

管理员表格比普通用户表格更高密度，但不能回到原始数据库表视觉。

渠道管理表格建议按区域组织：

```text
渠道信息
路由状态
健康状态
调用成本
结算状态
操作
```

表格规范：

- 5 - 8 行优先，避免首屏过密。
- 重要状态使用胶囊标签。
- 次级信息如渠道 ID、账号池、余额说明使用较小字号。
- 操作区优先使用图标按钮，配合 hover tooltip。
- 结算异常、余额不足、严重告警必须一眼可见。

状态标签建议：

```text
健康: 青绿 / 绿色弱背景
轻度告警: 橙色弱背景
严重告警: 红色弱背景
正常结算: 绿色弱背景
待结算: 橙色弱背景
结算异常: 红色弱背景
```

## 组件规范

### 筛选器

- 筛选器固定高度。
- 搜索框、下拉选择、日期范围、重置、查询保持一行布局。
- 筛选器不能挤占表格主要视野。

### 按钮

- 主按钮统一青绿色实心。
- 次按钮白底描边。
- 批量操作和导出操作靠近工具栏右侧。
- 高风险操作放入更多菜单或二次确认。

### 图标

- 图标线宽、圆角、尺寸必须统一。
- 管理端图标可更偏运营和治理，但不使用多套风格。
- 风险图标仅用于真正异常，不做装饰。

## 页面气质

管理员后台应更像成熟运营后台：

- 更高密度。
- 更强状态感。
- 更清楚的风险识别。
- 更快定位渠道、模型、结算和用户问题。

避免：

- 普通用户入口混入后台。
- 个人设置作为侧边栏菜单。
- 营销站导航。
- 大面积渐变和装饰性背景。
- 过细、过多、不可读的表格列。
- 管理后台和普通控制台复用同一套信息架构。
